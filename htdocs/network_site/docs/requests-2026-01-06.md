# Pedidos de 2026-01-06 (Network)

## Principais solicitações do dia
- Criar o sistema Network como site isolado, sem compartilhar banco/usuarios com o CRM.
- Usar banco parrudo (MySQL 8/InnoDB) para alto volume.
- Formularios e APIs devem ter validacao, consentimento, CSRF e mitigacao de spam (rate-limit).
- Capturar leads completos (PF/PJ: nome, email, telefone, empresa, CNPJ/CPF, regiao, area, objetivo, interesse, mensagem, CV/skills, consentimento, etc.).
- Landing deve exibir anuncios ativos vindos do banco e CTA para o formulario.
- Painel admin isolado: login com hash forte (Argon2ID), sessao, CSRF; listar leads, aprovar/negar, atribuir grupos; gerenciar anuncios (criar/editar/ativar/inativar).
- Script para criar admin inicial no banco.
- Orientacoes para MySQL robusto (buffer pool, SSD/NVMe, indices, conexoes, paginacao para volume alto).

## Itens implementados
- Backend isolado em `network_site` com MySQL (schemas: admins, leads, ads), CSRF, rate-limit, APIs JSON.
- Formulario publico completo e validado, gravando em MySQL.
- Landing com anuncios ativos do banco.
- Painel admin com login (Argon2ID), sessao/CSRF, listagem e acoes em leads (aprovar/negar, grupos) e anuncios (criar/editar/ativar/inativar).
- Script de seed de admin via MySQL.

## Pendencias operacionais
- Preencher credenciais reais do MySQL em `config/database.php`.
- Rodar o seed de admin apos o DB estar acessivel.
- (Opcional) Ativar conexao persistente, indices extras (ex.: regiao/area) e paginacao das APIs para volume muito alto.
